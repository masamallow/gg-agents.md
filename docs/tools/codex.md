# Codex CLI

OpenAI's terminal coding agent. Resources in this repo target `~/.codex/`.

## Layout in `~/.codex/`

| Path                 | Purpose                                          | Symlinked from         |
| -------------------- | ------------------------------------------------ | ---------------------- |
| `AGENTS.md`          | Global instructions (also project-level)         | `AGENTS.gg.md`         |
| `config.toml`        | Model, sandbox, approval, profiles, MCP servers ‚Äî see template below (not symlinked) | ‚Äî |
| `prompts/<name>.md`  | Custom slash prompts (`/<name>`)                 | `codex/prompts/`       |
| `auth.json`          | Auto-generated by `codex login` ‚Äî do not track   | ‚Äî                      |
| `history.jsonl`      | Session history ‚Äî do not track                   | ‚Äî                      |

## `config.toml` template

Codex CLI does not generally rewrite `config.toml`, but for consistency with the other tools it lives here as a copy-paste template rather than a symlink. Drop it into `~/.codex/config.toml` and tweak per machine.

```toml
# Codex CLI global configuration
# Reference: https://github.com/openai/codex/blob/main/docs/config.md

# Default model for new sessions.
model = "gpt-5-codex"

# Reasoning effort: "minimal" | "low" | "medium" | "high"
model_reasoning_effort = "medium"

# Approval policy: "untrusted" | "on-failure" | "on-request" | "never"
approval_policy = "on-request"

# Sandbox mode: "read-only" | "workspace-write" | "danger-full-access"
sandbox_mode = "workspace-write"

# Show reasoning summaries in the UI when the model emits them.
show_raw_agent_reasoning = false

# Keep `~/.codex/history.jsonl` enabled by default.
[history]
persistence = "save-all"

[sandbox_workspace_write]
# Network is off by default; flip per-project when truly needed.
network_access = false
# Extra writable roots beyond cwd and tmp (absolute paths).
writable_roots = []

# --- Profiles -----------------------------------------------------------
# Switch with `codex --profile <name>`. Profiles override top-level keys.

[profiles.readonly]
approval_policy = "on-request"
sandbox_mode = "read-only"

[profiles.yolo]
approval_policy = "never"
sandbox_mode = "danger-full-access"

# --- MCP servers --------------------------------------------------------
# [mcp_servers.example]
# command = "npx"
# args = ["-y", "@example/mcp-server"]
# env = { "API_KEY" = "..." }
```

## Key config knobs

- `model`, `model_reasoning_effort`
- `approval_policy`: `untrusted` | `on-failure` | `on-request` | `never`
- `sandbox_mode`: `read-only` | `workspace-write` | `danger-full-access`
- `[sandbox_workspace_write]` ‚Üí `network_access`, `writable_roots`
- `[profiles.<name>]` to override per-invocation (`codex --profile <name>`)
- `[mcp_servers.<name>]` for MCP integrations
